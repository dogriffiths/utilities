Core Principles

  • Chatbots are NOT deities; they are INTERNS
  • No prompt/response will be PERFECT
  • Ask for OPTIONS
  • Chatbots are good at QUANTITY; humans are good at QUALITY
  • CONVERSATIONS are more powerful than prompts
  • Use TRANSCRIPTIONS
  • Look for useful FILE FORMATS

The Chatbot Reality Check

Think of a chatbot as a really smart intern

  • Vanilla by default: Produces generic content without guidance
  • Requires oversight: Can't be left to do all your work
  • Needs verification: Always requires checking and validation
  • Structured guidance: Needs proper prompts to be effective

The PROMPT Framework

7 aspects of a prompt

  • ROLE: Point-of-view from which to respond (expert chef, consultant, etc.)
  • TASK: Clearly state what you want done
  • FORMAT: Specify how you want the output structured
  • TONE: Set the communication style (formal, informal, enthusiastic)
  • AUDIENCE: Define who this is written for
  • SCOPE: Set boundaries and constraints
  • STEPS-TO-INCLUDE: Specify what should be included

Repeat-and-Refine Technique

Start simple, then iterate

  1. Begin with a basic request
  2. Ask for variations or more options
  3. Refine based on results
  4. Add specific constraints or modifications
  5. Continue until you get exactly what you need

Project Scope Definition

Use a chatbot as a questioning consultant

  • Ask it to take you through discovery questions
  • Provide your answers to refine the scope
  • Request a summary of the defined scope
  • Use this as a foundation for story creation

Key benefit: Systematic approach to capturing requirements you might miss

Check Stories with INVEST

Help check the quality of a user story

  • Independent: Stories can be developed separately
  • Negotiable: Details can be discussed and refined
  • Valuable: Each story delivers user value
  • Estimable: Development effort can be assessed
  • Small: Manageable size for development
  • Testable: Clear success criteria can be defined

Acceptance Criteria

Transform stories into testable statements

  • Generate example true/false statements for each story
  • When all statements are true, the story is complete
  • Can generate example BDD scenarios from ACs
  • Scenarios can be used with tools like NotebookLM to build a knowledge base

Dependency Analysis

Useful when exporting stories to MS Project

  • Analyze logical dependencies between user stories
  • Output in project management formats (like "1FS" for Finish-to-Start)
  • Create CSV files for import into project tools
  • Warning: Results will need manual review and correction

Meeting Workflow

Transform recordings into actionable insights

  1. Record meetings (using tools like Whisper for transcription)
  2. Create outline of discussion topics
  3. Extract action items with owners and deadlines
  4. Identify key questions that were raised
  5. Summarize decisions made during the meeting

Email Composition Strategy

Use tent-pole sentences

  1. Write key thoughts in order in a text file
  2. Ask the chatbot to convert to email format
  3. Specify tone (celebratory, formal, urgent)
  4. Set word limits for conciseness
  5. Include all necessary context points

Writing Presentations

From idea to slides

  1. Identify purpose (reference, tutorial, pitch, story)
  2. Generate ideas using Aristotle's modes: Logic (data), Emotion (feelings), Behavior (actions)
  3. Create mindmap of key concepts
  4. Record yourself explaining the mindmap
  5. Transcribe and outline the recording
  6. Convert to Marp/Markdown format
  7. Transform to PPTX using tools like Pandoc

Feedback Analysis Tools

Turn qualitative feedback into quantitative insights

  • Sentiment Analysis: Convert text reviews to 0-10 scores
  • Categorization: Group feedback by themes (functionality, performance, etc.)
  • Net Promoter Score: Calculate from 0-10 ratings (Promoters: 9-10, Passives: 7-8, Detractors: 0-6)
  • Survey Design: Mix multiple choice, free text, and rating questions

Risk Management Framework

Systematic risk assessment

Content Processing Shortcuts

Quick content analysis

  • Outline creation: Summarize lengthy documents
  • Top points extraction: Get key takeaways
  • Box-outs: Quick summary panels (like this!)
  • Style transformation: Rewrite in different tones
  • Grammar correction: Fix errors and improve clarity
  • Translation: Convert to Plain English or other languages

General Tips

Practices for AI-powered project management

  • Always verify outputs: Chatbots make mistakes
  • Start broad, then narrow: Use the repeat-and-refine technique
  • Be specific about format: Request tables, bullets, or paragraphs as needed
  • Check for bias: Review suggestions critically
  • Protect confidential info: Don't use online tools for sensitive data
  • Combine with human judgment: Use AI as a starting point, not the final answer
