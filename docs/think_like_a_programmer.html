AI Agents: Think Like A Programmer

Lessons from a Defect Tracking System

Finding principles for building AI agents from the wisdom of famous programmers.

 

Core Principles for AI Agents

  • Use agents strategically - Don't replace entire systems; find key areas to apply them
  • Treat agents like people - Give them identities, roles, and proper oversight
  • Give humans oversight of agents - Agents excel at quantity; humans excel at quality
  • Use errors for training - Build knowledge bases from corrected outputs
  • Use baby steps - Start small, track error rates, scale gradually
  • Give agents an ID - Register with identity management for tracking and security

The Real Problem

Why the system actually failed

A car manufacturer built a defect tracking system that seemed too slow. But the real issue wasn't database performance or lack of tests.

  • The challenge: Graduate engineers spent significant time analyzing defects manually
  • The process: Dealerships, breakdown companies, and car telemetry reported defects
  • The bottleneck: Engineers created diagnoses (subsystem, component, severity) for quality dashboards
  • The insight: If built now, could have considered AI agents instead of just optimizing the existing system

Agents Are Different

Not recipes, but chefs

  • Recipes vs. Chefs: Traditional software follows exact steps; agents make decisions
  • Can make mistakes: Like humans, agents are fallible and need oversight
  • Still learning: The industry is still figuring out how to build agents effectively
  • Risky to go full replacement: Replacing entire systems with agents is too risky

Key insight: Use agents strategically in specific workflow areas, not as wholesale replacements

Strategic Agent Placement

Where to integrate agents in your workflow

  • Keep simple programs simple: If it's a basic recipe, leave it as traditional code
  • Target bottlenecks: Focus on time-consuming manual analysis tasks
  • Example solution: Replace manual engineer diagnosis with an AI agent
  • Implementation: Python agent reads defect database, creates diagnoses (subsystem, component, severity)

Remember: Identify key areas where agents add the most value

Security and Identity

Treating agents like employees

  • Register agents: Add them to your identity management system like new users
  • Identity tokens: Agents get tokens (like JWTs) to authenticate with tools
  • Tool restrictions: Some MCP servers can limit tool access based on agent identity
  • Controlled actions: Configure tools to restrict agent capabilities (e.g., email only to specific accounts)
  • Audit trail: Log and track agent actions using the same systems as humans
  • Threat detection: Security tools like Defender or CrowdStrike can monitor agent behavior

Core Learning Limitations

Why agents aren't experts yet

Michael Nygard (Release It): "Bugs will happen. They cannot be eliminated, so they must be survived instead."

  • General knowledge: LLMs have impressive general knowledge from internet training
  • Not specialized: Can't match specialists with decades of specific domain experience
  • Will make mistakes: Agents will produce incorrect diagnoses initially
  • Need supervision: Treat agents like interns requiring oversight and training

Human-Agent Hybrid

Combining quantity with quality

  • Complementary strengths: Agents = quantity, Humans = quality
  • Human oversight: Humans check subset of agent behavior
  • Certification process: Humans verify correct responses or fix incorrect ones
  • Knowledge base: Certified responses feed back into the system
  • Continuous improvement: Agent gets better as knowledge base grows
  • Bonus artifact: Creates a "golden set" for other applications

Example: Build "Professor Piston" chatbot from certified diagnoses for public car troubleshooting

Scaling with Quality

Kent Beck's wisdom applied to agents

Make it work â†’ Make it right â†’ Make it fast
  1. Start tiny: Begin with a small amount of data on day one
  2. Check heavily: Audit a large proportion of responses initially
  3. Correct errors: Keep correcting with small data volumes
  4. Build knowledge: Let the knowledge base grow until error rate drops
  5. Scale gradually: Increase volume only when error rate is acceptable
  6. Track metrics: Continuously monitor error rates as you scale

The Workflow Integration

How it all fits together

  1. Defects arrive from multiple sources (dealerships, telemetry, workshops)
  2. Agent creates diagnosis: subsystem, component, severity level
  3. Human engineers check subset of diagnoses
  4. Correct and incorrect diagnoses both get certified
  5. Certified diagnoses populate knowledge base
  6. Agent improves using knowledge base for future diagnoses
  7. Quality dashboard receives high-quality, high-volume diagnoses
  8. Production line and design team can act on reliable data

Quality Dashboard Impact

What the system enables

  • Heat maps: Identify which car parts have more problems
  • Production line: Modify manufacturing to design out defects
  • Design team: Fix component issues in next year's models
  • Financial impact: Defects are expensive to fix and affect sales
  • Speed advantage: Faster diagnosis means quicker response to problems
  • Volume advantage: Process more defects than human-only system

Key Takeaways

Summary of strategic agent deployment

  • Don't replace everything: Use agents strategically in specific workflow areas
  • Security matters: Give agents identities, restrict their capabilities
  • Expect mistakes: Plan for errors rather than hoping to eliminate them
  • Human + AI = Best: Combine agent quantity with human quality oversight
  • Learn from errors: Build knowledge bases from certified corrections
  • Start small: Use baby steps to scale while maintaining quality
  • Track everything: Monitor error rates and agent performance continuously
